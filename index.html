SAFEPOS · KTP Objective

The Problem Space · As I See It

01
Sensor Deception
In a metal-dense port environment, a sensor doesn't have to fail to cause a crash: it just has to be confidently wrong.
How does the system know when to stop trusting a sensor that hasn't gone offline, when it's just lying?
02
Weather as an Operational Gate
Port schedules don't pause for marginal conditions, and a grounded fleet is a liability. But so is a fleet that shouldn't have launched.
When conditions are marginal, how does the system decide whether to fly, and if it does, how does it stay safe?
03
Coordination Fragility & Contingency
Independent agents only stay safe when every neighbour's state can be trusted, yet not every mission ends cleanly.
If a neighbour is publishing a wrong position, how does the system detect that, stay safe, and what happens if a drone fails mid-operation over an active dock?
04
Access Control & Security
A twelve-drone fleet has twelve attack surfaces. Centralised control is a single point of failure; fully distributed control is ungovernable.
Across a twelve-drone fleet, who controls what, how do you prevent unauthorised access, and is one operator really meant to have their hands on everything?
These aren't edge cases. They are the design constraints.

Sensors plug in as adapters · not baked into the backbone. Any combination, any platform. Heterogeneous fleets supported by the architecture itself.
Classifier-Free Guidance
Sensor reliability is weighted at inference time. A noisy sensor gets downweighted · not trusted blindly. Directly extends the confidence-weighting logic from PROGENY.

Multipath interference or degraded signals detected · model continues on LiDAR and camera. Deployable adapters feeding neighbouring drone states enable real-time tactical deconfliction without retraining the core backbone.

08 / 11
Architecture Decisions

Teaching Drones to
Coordinate Without a
Central Controller

Each drone: own state + neighbour observations → independent policy → residual thrust + moment output
Policy Type
Independent per agent · no central controller. Centralised control is a single point of failure. Doesn't scale to a port fleet.
State Space
Own state + neighbour observations. Coordination emerges from training · not hard-coded rules that break on edge cases.
Action Space
Residual thrust + moment over a structured baseline. Better training stability. Maps cleanly to real actuator constraints.
Training Condition
Crate mass and size randomised throughout training. Variable payloads are the training condition · not an edge case the policy sees once.
